quali-fit — 개발 진행 & 로드맵

업무에 맞는 직원을 이유와 함께 추천하는 사내 웹 도구. 이 페이지는 비개발자용으로 우리가 일하는 법 · 지금까지 한 일 · 앞으로 할 일을 한눈에 정리합니다.

기준일: 2026-06-16 · 현재 버전 v0.0.13 · 깊은 기술 문서는 README, 살아있는 현황은 GitHub 이슈에 있습니다.

1우리가 일하는 법 (GitHub 협업 흐름)

새 기능 하나가 이런 5단계를 거쳐 실제 서비스에 올라갑니다. 비개발자도 이 흐름만 알면 진행 상황을 따라올 수 있어요.

① 이슈 (Issue)"무엇을 할지" 적는 할 일 카드
→
② 브랜치 (Branch)본문을 건드리지 않는 작업용 사본
→
③ PR (Pull Request)"이렇게 바꿨어요" 제안 + 리뷰
→
④ 머지 (Merge)공식본(main)에 반영
→
⑤ 자동 배포로봇이 서버에 올림 (~20초)

규칙: 버전 1개 = 이슈 1개 → 브랜치 → PR → 머지. 그래서 아래 "주차별 진행"의 각 항목에는 연결된 이슈/PR 번호가 붙습니다.
